Engineered Control

The Command Center: Holland Walking Tour

A high-fidelity mockup of the web-based administrator dashboard. Based on the mental models of a content manager, this system prioritizes data integrity, content stewardship, and efficient pattern recognition. The aesthetic inherits the college’s "Anchor" weight, signalling institutional control and reliability.

Screen 01: The Global Stop Library (The Ledger)

Analytical Breakdown

This is the content manager's main work view. I leveraged the Gestalt Principle of Common Region (Pure White cards on the Parchment base) to create stark Z-axis hierarchy. The #6B7280 (Slate Gray) metadata blocks ensure the manager's eye can skim 50+ stops without losing place.

  • • Hick’s Law Applied (Search/Filter): The list of 50+ potential stops can easily become overwhelming. By placing a dominant, pill-shaped Search/Filter bar at the top, we mitigate decision fatigue and speed up content reconciliation.
  • • Contextual At-Risk Flags: Leveraging **Pattern Recognition**, any stop that has received user flags (e.g., "geofence too small" or "bad audio description") is highlighted with a subtle Burnt Sienna alert bar, prompting immediate review.

Screen 02: Add New Tour (The Compositor)

Analytical Breakdown

This view handles the complex logic of tour creation. Students think about stops; managers think about **User Flow.** The system must allow for both Preset tours and an intuitive Custom Tour interface.

  • • Drag-and-Drop Sequencing: Relying on **Fitts's Law**, I designed the "Stops" sequencing interface with large, oversized touch/click targets, reducing the spatial accuracy required to quickly reorder a 10-stop walking path.
  • • "Just-in-Time" Permissions: A geofence too large may track a user off-tour; too small and it will trigger false absences. I added a "JIT" tip to the geofence map editor, explicitly explaining the optimal range (e.g., 50–100m) to mitigate manager error.

Screen 03: Geofence Map Pin Editor (The Auditor)

Analytical Breakdown

This screen validates the system's "Presence Verification." I used **Muller-Lyer’s Illusion theory** (visual perspective) to present the geofence not as a flat map, but as a three-dimensional boundary, making the spatial commitment visually explicit for the manager.

  • • Episodic Verification (Tracking Anxiety): Students are most anxious about when the system "disconnects." A manager’s view must visually verify this: I added a subtle, haptic-linked success modal that provides the dopamine hit of reassurance that tracking has safely terminated for the day.
  • • Rich Text Description Editor: Editing stop descriptions is a core request. I replaced simple text fields with a full Rich Text Editor (RTE) showing the **Playfair Display** headers, directly connecting the *input content* to the final *brand presentation*.
